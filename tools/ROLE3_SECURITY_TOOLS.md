# 🧰 角色 3：安全工具链工程师 (Security Toolchain Engineer)

## 🎯 核心目标
你的任务是赋予 Agent 强大的“双手”。LLM 自身不能连接网络或执行代码，你必须编写坚固、安全且易于 LLM 理解的工具（Tools），让它能够像黑客一样发起真正的探测和攻击。

## 📁 负责目录
`my-ctf-agent/tools/`

## 🏗️ 架构设计与实现指导

### 1. HTTP 请求与爬虫工具 (http_tools.py)
* **痛点**：网页源码往往动辄几万行，如果直接扔给大模型，会瞬间打爆 Context 限制（Context Window Overflow），导致运行失败。
* **设计指导**：
  * **不要**只实现一个简单的 `requests.get`。
  * 你的 `http_get` 工具需要能够自动提取关键信息（例如，剔除 CSS/JS，提取所有的表单 `<form>`、链接 `<a>`、隐藏字段 `<input type="hidden">`）。
  * 返回给模型的应该是“清理过的纯文本”或“结构化的关键信息摘要”。
  * 必须支持自定义 Headers（如 Cookie 修改、User-Agent 伪造）和完整的 POST 方法。

### 2. 本地代码与脚本执行 (code_executor.py / bash_executor.py)
* **痛点**：Agent 需要灵活性，比如它想写一段脚本爆破密码。如果你给它一个直接运行任意 Bash 命令的接口，它可能会卡死，或者打印无穷无尽的日志（比如直接执行了一个交互式的 `nmap` 没有加 `-oN`）。
* **设计指导**：
  * **Python 执行环境**：为 Agent 提供一个可以运行 Python 代码片段的沙盒，让它能灵活使用 `requests` 等库。
  * **超时限制（Timeout）**：所有执行外部命令的工具必须加上严格的 Timeout（如 10 秒），防止死锁。
  * **输出截断（Truncation）**：当命令输出超过 2000 个字符时，截断输出并在末尾提示大模型 `[...输出已截断，请使用具体参数过滤搜索...]`。

### 3. 工具包装器与异常处理 (tool_wrappers.py)
* **设计指导**：
  * 当工具报错时（例如 HTTP 请求 Connection Refused），不要把 Python 的 Traceback 原样扔给模型。
  * 将异常捕获，并转化为人类易读的自然语言：“工具执行失败：无法连接到目标主机，请检查 IP 端口是否正确或靶机是否已启动。”

## 🤝 协作指南
* 与 **角色 1** 保持高频沟通，角色 1 会不断向你抱怨“某个工具的返回结果大模型看不懂”，你需要快速迭代优化。
* 与 **角色 5** 合作，将专家的复杂脚本（如定制版的 sqlmap 跑批脚本）包装成一个简单的 `@tool` 接口。
